# External Integrations

**Analysis Date:** 2026-03-08

## APIs & External Services

**AI/LLM Providers:**
- Anthropic Claude - LLM chat via MCP chat service
  - SDK: `@ai-sdk/anthropic` ^3.0.47
  - Default model: `claude-opus-4-6`
  - Auth: `ANTHROPIC_API_KEY` (env var, used by AI SDK)
  - Implementation: `packages/backend/src/services/McpChatService.ts`

- Google Gemini - LLM chat via MCP chat service
  - SDK: `@ai-sdk/google` ^3.0.31
  - Default model: `gemini-2.5-flash`
  - Auth: `GOOGLE_GENERATIVE_AI_API_KEY` (env var, used by AI SDK)
  - Implementation: `packages/backend/src/services/McpChatService.ts`

- OpenAI - LLM chat via MCP chat service
  - SDK: `@ai-sdk/openai` ^3.0.33
  - Default model: `gpt-4o`
  - Auth: `OPENAI_API_KEY` (env var, used by AI SDK)
  - Implementation: `packages/backend/src/services/McpChatService.ts`

**Model Context Protocol (MCP):**
- MCP Server - Exposes Ralph orchestrator tools to AI models
  - SDK: `@modelcontextprotocol/sdk` ^1.27.1
  - Implementation: `packages/backend/src/mcp/RalphMcpServer.ts`
  - Tools exposed: start_loop, stop_loop, create_project, update_project, delete_project, kill_process, update_settings (and more)
  - Destructive tools require user confirmation via `McpChatService`

## Data Storage

**Databases:**
- SQLite via better-sqlite3
  - Connection: `RALPH_UI_DB_PATH` env var (default: `.ralph-ui/data.db`)
  - Client: Drizzle ORM (`drizzle-orm` ^0.45.1)
  - Schema: `packages/backend/src/db/schema.ts`
  - Migrations: `packages/backend/drizzle/` (generated by drizzle-kit)
  - Tables: projects, loop_runs, chat_sessions, chat_messages, notifications, settings
  - Connection management: `packages/backend/src/db/connection.ts`

**File Storage:**
- Local filesystem only (project paths, worktrees stored as text references in DB)

**Caching:**
- None (in-memory state via services)

## Authentication & Identity

**Auth Provider:**
- None - No user authentication
- Origin-based access control: `packages/backend/src/lib/origin.ts`
  - `isOriginAllowed()` - Validates request origins
  - `allowsDangerousOperations()` - Returns true when bind host is loopback

**Error Tracking:**

**Logs:**

## CI/CD & Deployment

**Hosting:**

**CI Pipeline:**
- Not detected in repository

## Communication Protocols

**WebSocket:**
- `@fastify/websocket` for real-time communication
- Implementation: `packages/backend/src/api/websocket.ts`
- Used for: Loop status updates, terminal I/O, monitoring events

**Server-Sent Events (SSE):**
- Custom SSE implementation for AI chat streaming
- Implementation: `packages/backend/src/app.ts` (`writeSseEvent` function)
- Endpoint: Chat stream responses

**tRPC:**
- Type-safe RPC over HTTP
- Server: `packages/backend/src/trpc/router.ts` via `@trpc/server/adapters/fastify`
- Client: `@trpc/react-query` in frontend

## External Process Management

**Ralph Binary:**
- External `ralph` CLI binary orchestrated by the backend
  - Resolution: `packages/backend/src/lib/ralph.js` (`resolveRalphBinary`)
  - Override: `RALPH_UI_RALPH_BIN` env var
  - Process management: `packages/backend/src/runner/ProcessManager.js`
  - Service: `packages/backend/src/services/RalphProcessService.ts`

**PTY Sessions:**
- node-pty for terminal session management
  - Service: `packages/backend/src/services/TerminalService.ts`
  - 30-second grace period before runtime cleanup

## Environment Configuration

**Required env vars (for AI chat features):**
- `ANTHROPIC_API_KEY` - For Claude models
- `GOOGLE_GENERATIVE_AI_API_KEY` - For Gemini models
- `OPENAI_API_KEY` - For OpenAI models

**Optional env vars:**
- `RALPH_UI_DB_PATH` - Database file location
- `RALPH_UI_BIND_HOST` - Server bind address
- `RALPH_UI_RALPH_BIN` - Ralph binary path override
- `PORT` - Backend server port
- `VITE_PORT` - Frontend dev server port
- `VITE_RALPH_ORCHESTRATOR_BACKEND_ORIGIN` - Backend URL for frontend

**Secrets location:**
- Environment variables (`.env.local` for local development)
