    Class `LoopGitService` with these methods:

    1. `async getCurrentBranch(cwd: string): Promise<string | null>` — Run `git rev-parse --abbrev-ref HEAD` in cwd. Return branch name or null on error.

    2. `async createBranch(cwd: string, loopId: string): Promise<{ branchName: string, baseBranch: string } | null>` —
       - Get current branch via `getCurrentBranch(cwd)`. If null (not a git repo), return null.
       - Generate branch name: `ralph/loop-${loopId.slice(0, 8)}` (first 8 chars of UUID).
       - Run `git checkout -b {branchName}` in cwd.
       - Return `{ branchName, baseBranch: currentBranch }`.
       - On error, log warning and return null (non-fatal — loop should still run without branch).

    3. `async pushBranch(cwd: string, branchName: string): Promise<{ pushed: boolean, reason?: string }>` —
       - First check if there are any commits to push by running `git log origin/{branchName}..{branchName} --oneline 2>/dev/null` or by checking if the remote branch exists. Simpler approach: just try to push.
       - Run `git push -u origin {branchName}` in cwd with a 60-second timeout.
       - Return `{ pushed: true }` on success.
       - On error, return `{ pushed: false, reason: stderr or error message }`.

    4. `async createPullRequest(options: { cwd: string, branchName: string, baseBranch: string, title: string, body: string, githubToken: string, owner: string, repo: string }): Promise<{ prUrl: string | null, reason?: string }>` —
       - Use `fetch` to POST to `https://api.github.com/repos/{owner}/{repo}/pulls` with:
         - `head: branchName`
         - `base: baseBranch`
         - `title`
         - `body`
         - Headers: `Authorization: Bearer {githubToken}`, `Accept: application/vnd.github+json`
       - Parse response: on 201, return `{ prUrl: response.html_url }`.
       - On 422 (often "A pull request already exists"), return `{ prUrl: null, reason: 'PR already exists or validation error' }`.
       - On other errors, return `{ prUrl: null, reason: error message }`.

    5. `async hasPendingChanges(cwd: string): Promise<boolean>` —
       - Run `git status --porcelain` in cwd.
       - Return `true` if output is non-empty.

    6. `async commitAllChanges(cwd: string, message: string): Promise<boolean>` —
       - Run `git add -A` then `git commit -m {message}` in cwd.
       - Return true on success, false on error (e.g. nothing to commit).

    - Test each method with mocked successful and failed git commands.
    - For `createPullRequest`, mock global `fetch`.
    - Follow existing test patterns: use `describe`/`it` blocks, `vi.fn()` for mocks.
